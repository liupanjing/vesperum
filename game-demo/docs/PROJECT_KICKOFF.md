# 项目启动记录

## 启动日期

2026-05-18

## 项目状态

正式立项，进入 Demo 原型开发阶段。

## 项目目标

制作 Vesperum 的 30-45 分钟垂直切片 Demo。第一章选择“玻璃镇”，用于验证核心机制是否成立。

## 第一阶段范围

必须包含：

- 玻璃镇基础数据。
- NPC 标签系统。
- 玩家可见显性数值。
- 玩家不可见群众账本。
- 选择结算系统。
- Boss 动态判定。
- 章节结算雏形。

暂缓包含：

- 完整战斗系统。
- 完整美术。
- 完整音乐。
- 多城镇主线。
- 商业化发布。

## 当前技术决策

- 引擎：Godot 4.x。
- 脚本：GDScript。
- 数据：第一阶段使用 JSON。
- 路径：项目工程放在 `game-demo/`。

## 重要设计决策

- 不做传统善恶值。
- 保留玩家可见的自我认知数值。
- 隐藏群众账本按 NPC 标签聚合。
- 武力不是坏数值，但代表恐惧与强力秩序。
- Boss 由玩家长期伤害、忽视、背叛或恐惧化的群体生成。
- 战斗系统需要重新设计，暂不采用“三槽 HP/Will/Fear”方案。

## 近期目标

1. 跑通纯数据选择原型。
2. 补齐玻璃镇 8-12 个 NPC。
3. 扩展到 6-10 个关键选择。
4. 完成 BossResolver 的首轮规则验证。
5. 重新讨论战斗系统方向。
