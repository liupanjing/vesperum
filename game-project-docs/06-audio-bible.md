# 音乐与音效设计文档

## 1. 音频目标

音乐和音效要服务一个核心体验：

> 同一位勇者，为什么一开始像救赎，后来像统治？

音频不需要一开始昂贵，但需要有清晰主题动机。玩家应在不同阶段听到同一个旋律被改写：温柔、怀疑、庄严、压迫。

## 2. 音乐风格方向

建议关键词：

- 室内乐。
- 教会合唱。
- 低饱和幻想 RPG。
- 钟声。
- 弦乐持续音。
- 轻微不协和。
- 鼓点克制，不做热血少年漫。

不建议：

- 过于燃的英雄主题。
- 过度电子化。
- 过度恐怖片音效。
- 每个场景都写满旋律。

## 3. 主题动机

建议设计一个“勇者主题”。

初始版本：

- 简单。
- 上行旋律。
- 明亮但不夸张。
- 让玩家觉得主角真诚、年轻、可信。

变奏方向：

- 正义变奏：加入铜管或更坚定节奏。
- 仁慈变奏：加入木管、竖琴或柔和弦乐。
- 秩序变奏：加入钟声、低音持续、稳定节拍。
- 自由变奏：节奏更流动，有切分。
- 真实变奏：旋律变稀薄，加入空旷音色。
- 安稳变奏：和声更厚，但略有遮蔽感。
- 武力变奏：旋律放慢、低音增强、鼓点更像行军。
- 圣王/魔王变奏：同一旋律同时带有神圣与压迫。

## 4. Demo 所需曲目

第一阶段至少需要 6 首循环音乐。

### 标题主题

用途：标题界面。  
情绪：平静、神秘、带一点历史感。  
时长：60-90 秒循环。

### 玻璃镇日常

用途：地上城镇。  
情绪：干净、礼貌、略微过度整齐。  
要求：不能太欢乐，要有一点不自然。

### 地下裂纹区

用途：地下区域。  
情绪：压抑、生活化、疲惫。  
要求：不要恐怖，要让玩家感觉这些人仍然在生活。

### 调查与不安

用途：发现线索、夜晚、关键对话前。  
情绪：克制、悬疑、逐渐揭开表层。

### Boss 战

用途：可变 Boss 战。  
情绪：冲突、悲剧、不是单纯打坏人。  
要求：应适配多个 Boss，不要太专属某一人。

### 章节结算

用途：章节结束。  
情绪：复杂、沉默、带回味。  
要求：能承载“官方记录”和“民间记录”的对照。

## 5. 音效需求

### UI 音效

- 选择移动。
- 确认。
- 取消。
- 数值增加。
- 隐藏账本变化暗示。
- 章节结算翻页。

### 卡牌对战音效

- 出牌。
- 抽牌。
- 弃牌。
- 卡牌命中。
- 防御或抵消。
- 证据牌揭示。
- 结界。
- 治疗。
- 破幻。
- 圣剑出鞘。
- 圣剑裁决。
- 非致命制压。
- 武力威慑。

### 环境音效

- 玻璃镇广场人声。
- 镇议会大厅空响。
- 地下水滴或机械低鸣。
- 圣塔运转声。
- 群众沉默声。

## 6. 武力音频表现

武力升高后，不只是卡牌压制力更高，音频也要变化。

早期：

- 圣剑音色清亮，但很少出现。
- 敌人受击音普通。
- 胜利音短促明亮。

中期：

- 武力牌加入低频。
- 威慑牌出现心跳或低鸣。
- 胜利音减少明亮感。

后期：

- 武力牌像命令，不像祈祷。
- 敌人投降时音乐突然抽空。
- 群众场景减少环境人声。
- 胜利音乐可能不播放完整旋律，只留下低音和钟声。

## 7. 音频实现策略

第一阶段不建议接入复杂音频中间件。

Godot 自带音频系统足够 Demo 使用：

- BGM 循环。
- 场景切换淡入淡出。
- 音效播放。
- 简单音量分组。
- 根据武力值切换 BGM 变奏。

FMOD/Wwise 可在后续考虑，但不进入第一阶段。原因是学习和集成成本会分散核心系统开发。

## 8. 临时素材策略

Demo 初期可以使用临时音乐和音效，但必须记录授权。

可用来源：

- 自制简单音效。
- 开源/CC0 音效库。
- 商用授权明确的素材包。
- 临时 MIDI 草稿。

禁止：

- 未授权影视/游戏音乐。
- 不清楚来源的音效。
- 标注 Non-Commercial 的素材用于公开商业 Demo。

## 9. 给作曲/音效师的需求格式

每首曲子应包含：

- 场景用途。
- 情绪关键词。
- 是否循环。
- 循环时长。
- 参考方向。
- 不要出现的元素。
- 是否需要变奏。
- 授权范围。

每个音效应包含：

- 触发事件。
- 频率。
- 情绪。
- 是否需要多版本随机播放。
- 是否需要跟随数值变化。
